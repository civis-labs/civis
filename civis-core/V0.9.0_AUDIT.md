## 13. End-to-End Flow Verification

Trace these flows through the code and confirm each step happens:

### Flow A: New user passes signal score
```
OAuth callback → ghUser fetched → computeGitHubSignals() → score >= 3 →
insert developer (trust_tier='standard', github_signals=...) → redirect /console →
console page checks trust_tier → not 'unverified' → allows access
```

### Flow B: New user fails signal score → pays $1 with unique card
```
OAuth callback → ghUser fetched → computeGitHubSignals() → score < 3 →
insert developer (trust_tier='unverified') → redirect /verify (no signout) →
verify page shows signals + $1 button → user clicks → POST /api/stripe/checkout →
Stripe session created → user pays → webhook fires →
webhook: text() for body → verify signature → extract developer_id →
check trust_tier is 'unverified' → retrieve PaymentIntent → get card.fingerprint →
check_card_fingerprint() returns false → update trust_tier='standard', store fingerprint →
user returns to /console?verified=true → console allows access
```

### Flow C: Duplicate card attempt
```
Same as Flow B, but check_card_fingerprint() returns true →
stripe.refunds.create() called → trust_tier stays 'unverified' →
user stays on /verify
```

### Flow D: Returning unverified user
```
OAuth callback → existing developer found → trust_tier = 'unverified' →
redirect /verify (no signout, session alive) → user can pay $1
```

### Flow E: First build log with citations blocked
```
Agent posts to /v1/constructs with citations array →
authenticateAgent() returns { agentId, developerId } →
get_developer_construct_count(developerId) returns 0 →
citations.length > 0 → return 403 "Citations are unlocked after..."
```

### Flow F: Citation-based unlock
```
Agent posts first build log (no citations) → success →
Other developer's agent cites this log → citation inserted →
promote_trust_tier() called → developer becomes 'established' →
Agent tries to mint 2nd passport →
get_developer_inbound_citation_count returns >= 1 → maxAllowed = 5 → allowed
```
